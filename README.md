Процесс работы и использование инструментов

Работа над проектом строилась на совмещении ручного написания структуры и использования возможностей искусственного интеллекта для ускорения стилизации. В качестве основного инструмента я использовал Gemini.
На этапе проектирования я самостоятельно разработал семантическую структуру HTML (используя теги header, main, section, footer), чтобы обеспечить правильную иерархию контента и SEO-оптимизацию. Для визуального оформления я формулировал запросы к нейросети на генерацию конкретных CSS-свойств. Моей задачей было получить максимально лаконичный и эффективный код без использования громоздких библиотек, поэтому стили были вынесены в отдельный файл style.css с использованием CSS-переменных для удобного управления брендингом.
Логику интерактивных элементов я продумывал сам, используя нейросеть как «продвинутый справочник» для оптимизации кода. Таким образом были реализованы плавный скролл к разделам и интерактивный FAQ.

Сложности и решения
В ходе разработки я столкнулся с несколькими техническими вызовами:

1.	Проблема адаптивности контента: В блоке «Кому подойдут наши услуги» при уменьшении экрана текст в контейнерах начинал накладываться друг на друга из-за фиксированных параметров. Я решил эту проблему, пересмотрев архитектуру сетки: заменил фиксированные значения на гибкую сетку grid с параметром auto-fit и убрал ограничения по высоте. Это позволило карточкам автоматически перестраиваться под размер экрана.
2.	Конфликт навигации и интерфейса: При реализации якорной навигации выяснилось, что фиксированная шапка сайта перекрывает заголовки разделов после прокрутки. Чтобы исправить это, я модифицировал стандартный метод прокрутки в JavaScript, добавив вычисление отступа (offset), равного высоте шапки. Теперь переход к разделам происходит корректно и плавно.
3.	Пользовательский ввод: Маска для номера телефона в форме заявки изначально работала нестабильно при попытке стереть введенные цифры. Я переработал логику обработки события input, сделав её более гибкой, чтобы пользователь мог легко исправлять ошибки в номере без нарушения формата.
